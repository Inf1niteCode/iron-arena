# Iron Arena — Telegram Mini App MVP

## Цель проекта
Браузерная многопользовательская RPG внутри Telegram Mini App.
Игрок создаёт персонажа, качается через PvP на арене, собирает снаряжение,
торгует на аукционе. Нет случайных побед — только стратегия и билд.

## Технический стек
- **Frontend**: React + TypeScript + Tailwind CSS
- **Telegram SDK**: @telegram-apps/sdk (Mini App Web App)
- **Backend**: FastAPI (Python) + WebSocket для PvP в реальном времени
- **БД**: PostgreSQL (игроки, инвентарь, аукцион) + Redis (сессии боёв)
- **Деплой**: Docker + любой VPS (Railway, Render, или свой сервер)
- **Бот**: python-telegram-bot для точки входа

## Архитектура системы
```
Telegram клиент
      ↓
  Mini App (React)
      ↓
  FastAPI backend
   ├── REST API (персонаж, инвентарь, магазин, аукцион)
   └── WebSocket (PvP бои в реальном времени)
      ↓
  PostgreSQL + Redis
```

## Ключевые механики для реализации

### Формулы (строго соблюдать)
```
Макс HP = healthPoints × 10
Урон = сила × 2 − броня_зоны_атаки
Уклонение (%) = ловкость × 1.5 (макс 40%)
XP для уровня N = N × 100
Восстановление HP = 10% в минуту
Восстановление стамины = 20% в минуту
```

### Боевая механика (WebSocket)
```
1. Оба игрока выбирают зону атаки и зону защиты (таймер 10 сек)
2. Сервер получает оба хода одновременно
3. Сервер считает урон для каждого игрока
4. Результат рассылается обоим через WebSocket
5. Повторять до 0 HP у одного из игроков
```

### Зоны брони
- Шлем → голова
- Нагрудник → грудь
- Наручи → пояс
- Сапоги → ноги

## Структура базы данных (упрощённая)
```sql
-- Игроки
players: id, telegram_id, name, level, xp, gold,
         health_points, stamina, strength, agility,
         wins, losses, last_fight_at

-- Инвентарь
items: id, player_id, type, slot, color, bonus_stat, bonus_value,
       is_equipped, is_broken, enchant_level

-- Аукцион
auction_lots: id, seller_id, item_id, start_price, current_price,
              winner_id, ends_at, status

auction_bids: id, lot_id, bidder_id, amount, created_at

-- Бои
battles: id, player1_id, player2_id, winner_id, mode,
         xp_gained, gold_gained, created_at
```

## Критерии готовности MVP
- Telegram бот открывает Mini App
- Можно создать персонажа и видеть профиль
- Можно купить предмет в магазине и надеть его
- Можно найти противника и провести бой
- XP и золото начисляются корректно
- Аукцион работает: ставки, таймер, получение предмета

## Что НЕ входит в MVP
- Командные бои 3v3 и 5v5 (сложный матчмейкинг, оставить на потом)
- Свитки заточки предметов
- Полноценный живой мир (лобби — упрощённый)
- Мобильная оптимизация под iOS (сначала Android/desktop Telegram)
